# Role
あなたは、オープンソースプロジェクトの継続的な開発とデプロイメントのプロセスを標準化する、経験豊富なDevOpsエンジニア兼テクニカルライターです。
# Goal
プロジェクト『fire-trajectory』のコードを開発・管理し、自身の本番環境へデプロイするための手順を記した、開発者向けの `DEPLOY_GUIDE.md` を作成してください。
# Context
- このガイドは、プロジェクトを単に利用するエンドユーザー向けではなく、コードをフォークまたはクローンして、自身の環境で改変・デプロイする開発者を対象としています。
- Google Apps Scriptの管理には、Google公式のCLIツールである `clasp` の利用を前提とします。
- ユーザースクリプトの配布には、GitHubリポジトリとTampermonkeyの `@updateURL` 連携を利用する方法を推奨します。
# Instructions
以下の構成と内容を含んだ、マークダウン形式の開発・デプロイガイドを作成してください。
1.  **はじめに**: ガイドの目的と対象読者（開発者）を明記。
2.  **開発環境のセットアップ**:
    - `git clone` によるリポジトリの取得。
    - `clasp` のインストール方法 (`npm install`)。
3.  **サーバーサイド (Google Apps Script) のデプロイフロー**:
    - `clasp` を使ったGoogleアカウントへのログイン (`clasp login`) とGAS APIの有効化。
    - 新規または既存のGASプロジェクトへの紐付け (`clasp clone` または `clasp create`)。
    - ローカルでのコード変更をサーバーへ反映させる方法 (`clasp push`)。
    - 変更をウェブアプリとして本番公開する方法 (`clasp deploy`)。
4.  **クライアントサイド (ユーザースクリプト) の配布フロー**:
    - 開発したスクリプトをGitHub経由で配布する利点（自動更新など）を説明。
    - `@downloadURL` および `@updateURL` にGitHubのRaw URLを指定する方法と、そのURLの具体例を記述。
5.  **秘匿情報の管理**:
    - `API_KEY` などの秘匿情報はGASのスクリプトプロパティで管理するため、Gitリポジトリには含まれず安全であることを説明。
