シミュレーション前提値（生活費、年金、退職金、インフレ率など）に既存実装とナラティブ層で差異が出た場合、**ナラティブ層を正とする**。

**Why:** ナラティブ層（別Claudeプロジェクト「資産運用計画」）で意思決定と前提条件の議論を継続中であり、こちらはあくまで数値計算側。前提が分岐すると判断材料の整合性が崩れる。

**How to apply:** Dashboard のデフォルト値や Simulation のロジックがナラティブ層と食い違っていることに気づいたら、ナラティブ層側に合わせて修正提案する。ユーザーがB列で上書きしている既存値は触らない（setupSimulation の既存値保持仕様を維持）。
