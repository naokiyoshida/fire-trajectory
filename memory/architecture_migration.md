2026-05-10、3層構成（Tampermonkey + PowerShell + GAS）から Playwright + Node.js + GAS に一本化することが決定された。一括移行（並行運用なし）。

**Why:** 既存3層構成は DOM 脆弱性、Chrome表示のちらつき、設定3箇所分散、テスト不可能、PC稼働必須など複数の弱点があった。ユーザーは「自動化重視・月次でOK・言語自由」を希望し、規約リスクと自動化の両立として Playwright によるローカル月次実行が最良と判断。

**How to apply:**
- 新コードは TypeScript + Playwright + googleapis (Sheets API) で書く
- 既存 `mf_sync_client.user.js` `RunSyncHidden.ps1` は Phase 7 で削除予定
- 既存 `gas_receiver_service.gs` は doPost / sync_data 系を削除し、`setupSimulation` と新規 `setupReports` のみ残す方向
- スケジュール: 月次1回（取引・資産まとめて）。バランス案ではなく最小頻度を選択
- 認証: Playwright storageState で Cookie 永続化。初回のみユーザーが手動ログイン (2FA含む)、以降ヘッドレス
- ローカル実行を起点とし、必要になればクラウド (Cloud Run / GitHub Actions) に同じコードで移行可能な設計とする
- 既存 GAS の Simulation シート構築ロジック（フィッシャー方程式、ライフイベント制御、月次360行）は温存

ナラティブ層に合わせる前提値（2026-05 家計入金モデルに刷新済み）:
- 基本生活費: 月35万
- 本人月収_家計入金: 月30万 / 本人ボーナス_年額_家計入金: 90万
- 配偶者月収_家計入金: 月13万 / 配偶者ボーナス_年額_家計入金: 0
- 本人年金: 月15万 (年180万) / 配偶者年金: 月6.5万 (年78万)
- 本人退職一時金: 300万 / 配偶者退職一時金: 0
- 想定利回り: 5%（名目）
- インフレ率: 2%（日銀目標 / 直近CPI 2.7〜3.2%。旧 0 から変更）

  実セル行は描画時に動的決定。

  （= `INDEX(設定!$B:$B, MATCH("key", 設定!$A:$A, 0))`）を使う。

FIRE必要資産ライン (2026-05-16 追加):
- シミュレーション I列「FIRE必要資産(目標年齢まで)」= 就労収入が途絶えても年金のみで
  FIRE必要資産_目標年齢 まで FIRE必要資産_目標残額 を下回らない最低期首資産。
  末尾の目標年齢月から `I(r)=I(r+1)/(1+実質月利)-年金(r)+支出(r)` で逆算。
  退職一時金は当てにしない（働き続けないと得られないため保守側）。
- 行数は `max(シミュレーション終了年齢, FIRE必要資産_目標年齢)` まで生成（逆算アンカー確保）。
- 最終行は下行参照を避けるため nextReq=目標残額 をハードコード（トリム後 #REF! 防止）。
- グラフは LINE 化し 期末資産(H) と FIRE必要資産(I) の2系列。交点＝FIRE可能時期。
- FIRE射程シートに「FIRE可能時期」(H>=I 最初の月) と「FIRE可能時の本人年齢」を追加。
  比較は ISNUMBER(I) の行のみ（目標年齢超の "" 行を除外）。
- 旧キー（本人手取り月収 / 配偶者年収_年額）は migration でサイレント破棄し新デフォルトで上書き。
  既存値の保持は項目名キーで突き合わせるため並び替えに非依存。
